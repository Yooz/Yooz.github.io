This blog and me

Github pages and Hexo

4天时间，从jekyll折腾到Hexo，算是尘埃落定了。之前的blog是在VPS上搭的LNMP+Wordpress，正如大多数人所吐槽的那样，Wordpress太大了，什么都为你准备好了，实际上可自己折腾的太少了(其实是你不喜欢PHP而已)，而Github Pages相比之下显得简洁、灵活、geek。而且我也不想把VPS仅仅用来搭个博客，既然github提供了免费的空间，而且社区又活跃，为什么不用呢。于是在一番搜索之后得到的结论是，在github在搭建自己的静态博客。我先是选择了jekyll，如果我没理解错的话，一个ruby写的静态页面生成器。之后又在领导桑的推荐下换成了Hexo，但是我发现两者其实并没有差别，只是用了不同的语言写的（后者用的NodeJS）。大概的步骤是：

  • 我们在本地用Markdown写好页面
  • 通过Hexo（jekyll）渲染，生成HTML
  • 然后部署到Github Pages上

（如果此处理解有误，还请指出）

关于如何在Github上用Hexo搭建博客，我想你只要在搜索引擎里面里面输入github, hexo, 博客三个关键字，就会有各种各样的教程返回，所以在此我也不想多说。如果遇到什么问题，我倒是乐于帮忙，因我我当时也遇到了一些问题。总之，这个博客的初衷就是：分享知识。搭建这样一个博客能提高的技能有：

  • Shell 如果你使用UNIX或者Linux操作系统
  • Git 如果你有Github，是开源爱好者并且乐于分享
  • Markdown 如果你不堪忍受Word的排版

成为一个不愚蠢的人

成为一个不愚蠢的人

起先，我想把标题起为“成为一个相对聪明的人”，但是想了想觉得不妥。首先，“相对”是一种不确定性的说法，而“聪明”也无从定义，那么“不愚蠢”就是一个尽可能客观的说法了。事实上，成为一个不愚蠢的人也很困难。如果你认真审查这个世界，你会发现90%的人都是愚蠢的，说这种话有点“纳粹精英主义”的嫌疑。但在我看来，事实就是如此。很多时候，我都会觉得自己是个彻头彻尾的蠢货，反过来，我也为自己能意识到这一点感到庆幸。

所以我想成为一个不愚蠢的人，不愚蠢的人是什么样的 — 有明确的人生目标，清晰的人生规划，理性的思维以及坚定的执行力。当然，这只是我价值观里的不愚蠢的人，我的身边有不少这样的人，这样的结论也是我通过观察他们的行为得出来的。环境对人的影响是巨大的，美国某杂志有篇文章研究了”peer pressure“对人的影响，对它的结论我深信不疑，为了不和蠢货打交道，最好去一个大家都不蠢的环境中。

远离社交网络

同样，那篇文章中也提到了社交网络的影响，流言在社交网络上肆意传播，而不同成长环境，不同教育背景，不同人生经历的人分辨信息的能力是不同的。为了是自己不变的那么愚蠢，一个不错的办法是远离社交网络。离开了社交网络，真正关心你的朋友仍会关心你，而对于无关紧要的人，也不会有任何影响。远离社交网络和使用社交工具并不是冲突的，它们是不同的概念。

我更倾向于把一段时间的经历做一个总结，就想现在这样，而不是把一些鸡毛蒜皮的事都展现在社交网络中，后者会引起大部分人的反感。在写文章的时候，我会时刻反驳自己的观点，经过推理和反思，以确保我的观点具有“时限性“的说服力。原因在于，说不定几年后的自己会感概，“我原来说过那么傻逼的话”。

专注比努力和认真重要

“认真”“努力”和“专注”，这三个词都是来自于大学里一个我非常佩服的同学的签名。事实上，他也做到了。一开始，我并没有在意，直到我看到过结果，他获得了Google的工作。结果是最能说明这条理论是否适用的标准。对于我来说，专注是我最缺少的一种品质。努力和认真，我是可以做到的，实际我也做到了。我明白自己的问题在哪里，我是这样一个人，对于感兴趣的事情，我会想法设法完成它，结果就是我确实完成了。但是完成之后，我就失去了对这件事情的兴趣，转而投向下一个目标。这就类似与BFS还是DFS，大部分情况下，BFS比DFS好用，但是，困难的问题，都是DFS解决的。而且，在学术科研领域，更需要的就是专注。我会努力训练这方面的能力。

保持清醒保持孤独

2014年2月-2015年2月，这一年的时间我思考了很多，也实践了很多。保持清醒和孤独，能让我时刻反思自己。对于未来，我却还是不敢明确，在结果没有出来之前，再多的假想都是多余。

机器学习(1) -- 回归问题

回归问题

线性回归问题—最小二乘法.

在机器学习算法中，通常可以分为回归、聚类、贝叶斯、决策树等问题。而在回归问题中最典型的就是线性回归(Linear Regression)，线性回归中常用的方法是最小二乘法(Ordinary Least Square).本篇文章将详细介绍最小二乘法的推导过程，以及应用最小二乘法进行线性回归的例子。

线性回归属于监督性学习，所谓监督性学习，是指给出一定的训练样本，通过选定特定的机器学习算法，构造假设函数h，根据训练样本求得函数h的参数。那么对于需要解决的问题输入x，经过函数h处理，我们可以得到预测的结果。如下图：

问题的关键在于怎么选择学习算法，构造h函数。对于线性回归问题来说，通常我们有一组变量，用一个列向量XT(或者tX)表示，那就是：

    tX = [x0, x1, x2, ... ,xn]      // 通常 x0 = 1

训练集中的每个样本都是这样的一个一维的向量，对应的参(系)数向量假设为C:

    C = [c0, c1, c2, ..., cn]

根据线性回归的定义，我们假定的h函数就为h(x;c) = CTX(此处不明白h这种表示方法的可以看一下概率论的参数估计那一章)

    h(x;c) = c0 + c1*x1 + c2*x2 + ... + cn*xn

我们要明白回归问题实质上是对于输入的X，尽可能给出准确的预测。事实上，最小二乘法就是这样一个方案，来使得对线性回归的问题预测最优。对于样本集中的每个xi，都有与之对应的yi。最小二乘法就是找出这样一组参数，满足:

How do I learn Machine Learning

How do I learn Machine Learning?

很早在quora上看到了这个问题，现在有空了，就把这些整理一下，当然要真正掌握Marchine Learning，还是得经过科班训练才行。从MIT，Stanford…到国内的高校，ML都已经成为热门的研究方向。事实上，学术界的热度并不高于工业界，随便上LinkedIn或者国内的招聘网站上搜索ML、DM的职位，需求量仍旧很大。随着像Google，Facebook，国内BAT这些公司“大数据“的产生，不可避免地推动着学术界的研究。

当然对于一个ML的初学者，扎实的数学功底是必须的，特别是概率统计和线性代数。这里可以提供一些资料：

  • Linear Algebra and Online lecture
  • Probability and Statistics , Probability Theory or Elements of Statistical Learning
  • Multivariable calculus or this calculus and Optimization

总之，概率统计，线性代数和多元积分学等基础知识要掌握。接着就是被大家广为熟知的Andrew Ng的机器学习公开课 或者另一个机器学习公开课以及NLP。

学习理论是不够的，各路大牛们都在推荐kaggle竞赛。这里有几个相关的推荐[1][2]。知乎上也有人回答了kaggle比赛的收获和细节。

以我目前的认识水平，显然不能再进行下一步了。等我完成了以上训练，或者对机器学习有更加清晰的认识之后，我会继续更新下去。
