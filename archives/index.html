This blog and me

Github pages and Hexo

4天时间，从jekyll折腾到Hexo，算是尘埃落定了。之前的blog是在VPS上搭的LNMP+Wordpress，正如大多数人所吐槽的那样，Wordpress太大了，什么都为你准备好了，实际上可自己折腾的太少了(其实是你不喜欢PHP而已)，而Github Pages相比之下显得简洁、灵活、geek。而且我也不想把VPS仅仅用来搭个博客，既然github提供了免费的空间，而且社区又活跃，为什么不用呢。于是在一番搜索之后得到的结论是，在github在搭建自己的静态博客。我先是选择了jekyll，如果我没理解错的话，一个ruby写的静态页面生成器。之后又在领导桑的推荐下换成了Hexo，但是我发现两者其实并没有差别，只是用了不同的语言写的（后者用的NodeJS）。大概的步骤是：

  • 我们在本地用Markdown写好页面
  • 通过Hexo（jekyll）渲染，生成HTML
  • 然后部署到Github Pages上

（如果此处理解有误，还请指出）

关于如何在Github上用Hexo搭建博客，我想你只要在搜索引擎里面里面输入github, hexo, 博客三个关键字，就会有各种各样的教程返回，所以在此我也不想多说。如果遇到什么问题，我倒是乐于帮忙，因我我当时也遇到了一些问题。总之，这个博客的初衷就是：分享知识。搭建这样一个博客能提高的技能有：

  • Shell 如果你使用UNIX或者Linux操作系统
  • Git 如果你有Github，是开源爱好者并且乐于分享
  • Markdown 如果你不堪忍受Word的排版

How do I learn Machine Learning

How do I learn Machine Learning?

很早在quora上看到了这个问题，现在有空了，就把这些整理一下，当然要真正掌握Marchine Learning，还是得经过科班训练才行。从MIT，Stanford…到国内的高校，ML都已经成为热门的研究方向。事实上，学术界的热度并不高于工业界，随便上LinkedIn或者国内的招聘网站上搜索ML、DM的职位，需求量仍旧很大。随着像Google，Facebook，国内BAT这些公司“大数据“的产生，不可避免地推动着学术界的研究。

当然对于一个ML的初学者，扎实的数学功底是必须的，特别是概率统计和线性代数。这里可以提供一些资料：

  • Linear Algebra and Online lecture
  • Probability and Statistics , Probability Theory or Elements of Statistical Learning
  • Multivariable calculus or this calculus and Optimization

总之，概率统计，线性代数和多元积分学等基础知识要掌握。接着就是被大家广为熟知的Andrew Ng的机器学习公开课 或者另一个机器学习公开课以及NLP。

学习理论是不够的，各路大牛们都在推荐kaggle竞赛。这里有几个相关的推荐[1][2]。知乎上也有人回答了kaggle比赛的收获和细节。

以我目前的认识水平，显然不能再进行下一步了。等我完成了以上训练，或者对机器学习有更加清晰的认识之后，我会继续更新下去。
