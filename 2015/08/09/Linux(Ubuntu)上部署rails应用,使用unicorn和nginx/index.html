Linux(Ubuntu)上部署rails应用,使用unicorn和nginx

Posted on 2015-08-08   |   In learn   |  

这篇文章算是做一个总结吧，我就会一步步地列出在Linux服务器上怎么搭建一个rails应用。我使用的服务器是阿里云的VPS。配置如下

  • CPU： 1核
  • 内存： 1024 MB
  • 操作系统： Ubuntu 14.04 64位
  • 带宽：1Mbps

安装必要的包、软件、以及数据库

安装git，使用git下载

我是通过在本地写好了rails app，再将app上传到github，然后在我的服务器中把app clone下来。在服务器上进行部署的，所以首先我会介绍一下git的基本用法。（不是通过这种方法的可以跳过这里）

  1. ssh连到服务器, ssh root@serveraddress
  2. 在服务器上安装git. apt-get install git
  3. 生成ssh key. ssh-keygen 然后.ssh目录下会生成id_rsa. cd ~/.ssh
  4. 登陆你的github(如果没有需要申请)，在setting -> SSH keys那里添加(Add ssh key)id_rsa的内容。
  5. 把你的项目clone下来.git clone git@github.com:username/appname cd appname && git pull
  6. 修改目录权限chmod 777 appname

安装MySQL数据库

我的项目中一般都是使用MySQL作为关系型数据库。如果需求安装其他数据库，请查看相关官方文档说明。

  1. 首先更新一下包 apt-get apt-get update
  2. 下载MySQL服务器 apt-get install mysql-server和apt-get install libmysqlclient-dev 过程中会让你输入密码，如果不输入，密码默认为空。
  3. 登陆数据库查看一下。mysql -u root -p 输入密码。如果成功登陆，说明数据库下载成功。
  4. 配置数据库。默认情况下sock会在/tmp/这个目录下。其他配置参考MySQL官方文档。

下载安装Ruby和Rails

对于ruby的包管理，我推荐使用rvm。首先安装rvm，再安装ruby，再安装rails。通过rvm来管理ruby和rails的版本。

安装rvm、ruby以及rails

apt-get install curl
\curl -sSL https://get.rvm.io | bash -s stable
source ~/.profile
rvm -v
rvm install 2.0.0
rvm use 2.0.0
gem install rails -v 4.2.1 --no-ri --no-rdoc
rvm gemset create rails421
rvm use 2.0.0@rails421

请确保上面每一个步骤都是成功的。curl是一个catch url工具，rvm是ruby的版本管理工具。rails是ruby的一个gem包，gem是ruby的包管理工具。最后在你的服务器上试一下

rvm -v
ruby -v
rails -v

是否显示出对应的版本。如果正确，说明安装成功。

配置rails 应用的数据库和unicorn

对于数据库的配置，在/appname/config/database.yml中进行配置。

对于unicorn，在Gemfile中加入gem ‘unicorn’然后创建unicorn.rb文件并在/appname/config/unicorn.rb中进行配置。一般为

# set path to application
# app_dir is your rails app directory path
app_dir = File.expand_path("../..", __FILE__)
shared_dir = "#{app_dir}/shared"
working_directory app_dir


# Set unicorn options
worker_processes 2
preload_app true
timeout 30

# Set up socket location
listen "#{shared_dir}/sockets/unicorn.sock", :backlog => 64

# Logging
stderr_path "#{shared_dir}/log/unicorn.stderr.log"
stdout_path "#{shared_dir}/log/unicorn.stdout.log"

# Set master PID location
pid "#{shared_dir}/pids/unicorn.pid"

创建目录mkdir -p shared/pids shared/sockets shared/log

最后执行bundle install

下载并配置nginx

nginx是一个反向代理服务器，也是目前最受欢迎的开源代理服务器。
首先下载nginx并进行配置。它的配置在/etc/nginx/这个目录下

apt-get install nginx
cd /etc/nginx/sites-available
vi default

对这个文件进行编辑

upstream app {
# Path to Unicorn SOCK file, as defined previously
server unix:/path/to/app/shared/sockets/unicorn.sock fail_timeout=0;
}


server {
    listen 80 default_server;
    # listen [::]:80 default_server ipv6only=on;

    root /path/to/app/public;
    index index.html index.htm;

    # Make site accessible from http://localhost/
    server_name localhost;
    location @app {
            proxy_pass http://app;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header Host $http_host;
            proxy_redirect off;
    }

    location / {
            try_files $uri/index.html $uri.html $uri @app;
            # proxy_pass http://127.0.0.1:3000
            # First attempt to serve request as file, then
            # as directory, then fall back to displaying a 404.
            # try_files $uri $uri/ =404;
            # Uncomment to enable naxsi on this location
            # include /etc/nginx/naxsi.rules
    }
    error_page 500 502 503 504 /500.html;
    client_max_body_size 4G;

注意：/path/to/app/ 指的是你的应用的路径

chmod g+x /username && chmod g+x /username/appname && chmod g+x /username/appname/public

配置完之后启动unicorn和nginx

bundle exec unicorn -c config/unicorn.rb -p 3000 -D
service nginx restart

最后，你就可以通过ip访问页面了。
