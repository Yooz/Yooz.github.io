程序员面试中遇到的有趣的技术问题

原文来自Quora的一个问题，我觉得很有趣，很多题目很新颖，也能考验一个人的基本功和思维能力。现在把它翻译成中文，并且给出我的个人答案(不一定最好，欢迎讨论)。

  • 给出一个unsigned int，用高效的方法求出它的比特数。比如:

    2(10) = 1
9(1001) = 2
13(1101) = 3
    我的思路是，通过不断除2，模2判断余数是否为1，如果为1计数器就增加。 
    int countSetBits(unsigned int num){
      int n = num;
      int count = 0;
      while (n){
        if (n % 2 == 1)
          cout++;
        n /= 2;
      }
      return count;
}
    这算法不够高效，既然是对一个二进制的数求比特数，用二进制的操作符更好。于是改进算法，通过每次向右移位(相当于除2)，再通过和1进行'与'运算判断最右一位是否为1。 
    int countSetBits(unsigned int num){
      int n = num;
      int count = 0;
      while (n) {
        count += n & 1;
        n >>= 1;
      }
      return count;
}
    以上两种方法是我能想出来的方法了，然后我在答案中看到了另一种解法叫做: Brian Kernighan’s Algorithm。 
    int countSetBits(int num){
    int n = num;
    int count = 0;
    while (n){
      n &= (n - 1);
      count ++;
    }
}

    if (\)

  • 仅允许修改或者增加下面代码中的一个字符，使得程序打印20次*

    int main()
      {

      int i, n = 20;
  for ( i = 0; i < n; i--)
      printf("\*");
  return 0;

    }
    见过多次的题了。
    方法1： for循环里面最后一个i改成n，也就是 n—

    方法2： for循环里面第二个i前面加上负号，也就是 -i < n

    方法3： for循环里面第二个条件的<改为+，也就是 i + n

    欢迎补充

  • 设计一个算法逆序一个句子中的所有单词。比如”Today is a nice day”变成”day nice a is Today”.

    很老的题，见过很多次了。

    我的想法就是先逆序整个句子，再分别逆序每个单词。[A B C]-1 = [C-1 B-1 A-1]，再对每个子矩阵求逆就变成[C B A]了。

    我的想法就是用二分法，虽然这样有可能只试得到两次。然后看到有个解法.
